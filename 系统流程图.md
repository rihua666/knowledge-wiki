# 技能广场 — 系统流程图

> 版本：v1.1 | 日期：2026-04-15 | 状态：待确认

## 1. 系统角色与边界

> 技能广场作为技术门户系统内的功能模块，所有用户均已通过技术门户完成登录，无需在本模块内处理登录流程。

```
┌─────────────────────────────────────────────────────────┐
│                    技能广场 (Skill Hub)                    │
│                                                          │
│  ┌──────────────────────┐  ┌────────────────────────┐    │
│  │      普通用户         │  │       管理员           │    │
│  │                      │  │                        │    │
│  │ · 浏览 Skill         │  │ · 普通用户全部权限      │    │
│  │ · 搜索 / 筛选        │  │ · 审核 Skill           │    │
│  │ · 上传 Skill         │  │ · 下架 Skill           │    │
│  │ · 修改自己的 Skill    │  │ · 管理分类标签         │    │
│  │ · 下载 Skill         │  │                        │    │
│  │ · 查看审核状态       │  │                        │    │
│  └──────────────────────┘  └────────────────────────┘    │
│                                                          │
│  外部依赖:                                                │
│  · 内部 LLM API (AI 预审)                                 │
│  · Git 仓库服务 (Git 导入)                                │
└─────────────────────────────────────────────────────────┘
```

**角色说明：**

| 角色   | 权限                             | 说明         |
| ---- | ------------------------------ | ---------- |
| 普通用户 | 浏览、搜索、上传、下载、修改自己的 Skill、查看审核状态 | 系统默认角色     |
| 管理员  | 普通用户全部权限 + 审核、下架、管理分类标签        | 由系统分配的管理权限 |

## 2. 核心业务流程图

```mermaid
flowchart TD
    Start([用户进入技能广场]) --> Home[首页]

    %% 浏览与搜索
    Home --> Search[搜索 / 筛选 / 分类浏览]
    Search --> ViewDetail[查看 Skill 详情]
    ViewDetail --> Download[下载 Skill ZIP]

    %% 上传流程
    Home --> Upload[上传 Skill]
    Upload --> UploadChoice{上传方式}
    UploadChoice --> ZIP[上传 ZIP 压缩包]
    UploadChoice --> Git[填写 Git 仓库地址]

    ZIP --> Parse[解析 SKILL.md + 文件结构]
    Git --> Clone[Clone 仓库] --> Parse

    Parse --> AIPreview[AI 预审]
    AIPreview --> AICheck{AI 预审通过?}

    AICheck -- 不通过 --> NotifyReject[通知用户: 不通过 + 原因]
    NotifyReject --> Modify[用户修改 Skill]
    Modify --> AIPreview

    AICheck -- 通过 --> PendingReview[进入待审核队列]
    PendingReview --> AdminReview[管理员审核]

    AdminReview --> AdminDecision{管理员决定}
    AdminDecision -- 通过 --> Published[发布上线]
    AdminDecision -- 不通过 --> NotifyReject

    %% 修改已发布的 Skill
    Published --> UpdateRequest[用户发起修改]
    UpdateRequest --> AIPreview
```

## 3. AI 预审流程

```mermaid
flowchart TD
    SkillUploaded[Skill 提交] --> ValidateFormat[格式校验]
    ValidateFormat --> FormatOK{SKILL.md 格式合法?}

    FormatOK -- 否 --> FormatReject[拒绝: 格式不合法]
    FormatReject --> NotifyUser[通知用户修正]

    FormatOK -- 是 --> SecurityCheck[AI 安全审查]
    SecurityCheck --> SecurityOK{包含敏感内容?}
    SecurityOK -- 是 --> SecurityReject[拒绝: 安全风险]
    SecurityReject --> NotifyUser

    SecurityOK -- 否 --> QualityCheck[AI 质量评估]
    QualityCheck --> QualityReport[生成质量报告]

    QualityReport --> QualityScore{质量评分 >= 阈值?}
    QualityScore -- 否 --> QualityReject[退回: 质量不达标 + 改进建议]
    QualityReject --> NotifyUser

    QualityScore -- 是 --> SimilarCheck[查重检测]
    SimilarCheck --> SimilarResult{相似度 > 阈值?}
    SimilarResult -- 是 --> SimilarWarn[标记相似 Skill + 提交管理员判断]
    SimilarWarn --> AdminQueue

    SimilarResult -- 否 --> AdminQueue[进入管理员审核队列]
    AdminQueue --> AdminReview[管理员查看 AI 预审报告 + 原始内容]
    AdminReview --> FinalDecision[最终审核决定]
```

### AI 预审检查项

| 检查阶段 | 检查内容 | 不通过处理 |
|----------|----------|-----------|
| 格式校验 | SKILL.md 是否存在、YAML frontmatter 是否完整、必填字段（name/description） | 自动拒绝，通知修正 |
| 安全审查 | 是否包含敏感信息（API Key、密码、内部 IP）、是否包含恶意指令 | 自动拒绝，标记风险类型 |
| 质量评估 | 描述清晰度、指令完整性、示例充足度、格式规范性 | 退回修改 + AI 改进建议 |
| 查重检测 | 与已有 Skill 的名称/描述相似度 | 标记相似项，交管理员判断 |

## 4. Skill 数据生命周期

```mermaid
stateDiagram-v2
    [*] --> 草稿: 用户保存未提交
    草稿 --> 待AI预审: 用户提交
    待AI预审 --> AI预审通过: AI 审查通过
    待AI预审 --> 预审退回: AI 审查不通过
    预审退回 --> 待AI预审: 用户修改后重新提交
    AI预审通过 --> 待管理员审核: 进入审核队列
    待管理员审核 --> 已发布: 管理员通过
    待管理员审核 --> 审核退回: 管理员退回
    审核退回 --> 待AI预审: 用户修改后重新提交
    已发布 --> 待AI预审: 贡献者提交修改
    已发布 --> 已下架: 管理员下架
    已下架 --> 待AI预审: 重新上架申请
```

### 状态说明

| 状态 | 说明 | 可见性 | 可执行操作 |
|------|------|--------|-----------|
| 草稿 | 用户未提交 | 仅上传者本人 | 编辑、提交审核、删除 |
| 待 AI 预审 | 已提交，AI 审查中 | 仅上传者本人 | 等待（不可编辑） |
| 预审退回 | AI 审查不通过 | 仅上传者本人 | 查看报告、修改后重新提交 |
| 待管理员审核 | AI 通过，排队中 | 仅上传者本人 | 等待（不可编辑） |
| 审核退回 | 管理员退回修改 | 仅上传者本人 | 查看意见、修改后重新提交 |
| 已发布 | 审核通过，公开可见 | 所有用户 | 下载、上传者可发起修改 |
| 已下架 | 管理员下架 | 仅上传者和管理员 | 申请重新上架 |

## 5. 页面结构总览

```
技能广场
│
├── 首页 (/)
│   ├── 导航栏（logo、导航菜单）
│   │   ├── [按钮] 上传 Skill → 弹出上传表单
│   │   └── [按钮] 我的 Skill → 弹出我的技能列表
│   ├── 搜索栏 + 筛选器（分类/标签/排序）
│   └── 全部 Skill 列表（按上传时间正序排列，点击新窗口打开详情页）
│
├── Skill 详情页 (/skill/:id) [新窗口打开]
│   ├── 基本信息（名称、标签、贡献者、版本、下载量）
│   ├── Skill 内容（SKILL.md 渲染）
│   ├── 文件结构树
│   ├── 使用指南
│   └── 下载按钮
│
└── 管理后台 (/admin) [仅管理员]
    ├── 审核队列（待审核列表 + AI 预审报告）
    ├── 审核操作（通过/拒绝）
    ├── 已发布 Skill 管理（下架）
    └── 统计概览
```

### 页面交互说明

| 交互 | 触发方式 | 目标 |
|------|----------|------|
| 打开上传弹窗 | 点击顶部"上传 Skill"按钮 | 弹出上传表单 |
| 打开我的 Skill | 点击顶部"我的 Skill"按钮 | 弹出我的技能列表 |
| 打开详情页 | 点击首页技能列表项 | 新窗口打开详情页 |
| 排序切换 | 选择排序下拉框 | 按上传时间/下载量/名称排序 |

## 6. 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | React 18 + TypeScript + Vite + Tailwind CSS | SPA 单页应用 |

| 数据库 | MySQL 8.x | 关系型存储 |
| 文件存储 | 本地 / MinIO | Skill 文件和 ZIP 包存储 |
