# 项目说明文档生成要求

本文档定义了生成项目说明文档的标准规范。

---

## 一、文档基本信息

### 必需字段

| 字段 | 说明 | 示例 |
|------|------|------|
| 标题 | 项目名称 + "说明文档" | DeerFlow 说明文档 |
| 项目地址 | GitHub 仓库地址 | https://github.com/bytedance/deer-flow |
| 版本 | 当前版本号 | 2.0 |
| 许可证 | 开源许可证 | MIT License |
| 简介 | 1-2 句话项目描述 | 开源的 super agent harness 框架 |

---

## 二、文档目录结构

必须包含以下章节（根据项目类型可调整顺序）：

1. **目录** - 自动生成的可点击目录
2. **项目概述** - 项目简介、关键特性
3. **快速开始** - 环境要求、安装步骤、访问地址
4. **核心功能** - 主要功能点列表
5. **技术架构** - 技术栈、依赖管理
6. **模型配置** - 支持的模型及配置示例
7. **本地服务** - 服务地址列表（表格）
8. **Python Client 调用** - SDK 使用示例
9. **REST API 调用** - API 请求示例
10. **Skill 编写指南** - 自定义扩展指南
11. **配置文件详解** - 配置项说明
12. **项目文件结构** - 目录树
13. **常用命令** - Makefile/脚本命令
14. **故障排查** - 常见问题解决
15. **安全使用建议** - 安全注意事项
16. **相关文档** - 链接到官方文档

---

## 三、内容规范

### 快速开始

必须包含：
- 环境要求（Python 版本、Node 版本、依赖工具）
- 安装步骤（克隆 → 配置 → 安装 → 启动）
- 服务访问地址

### 核心功能

使用 Markdown 列表格式，每个功能包含：
- 功能名称
- 功能描述
- 适用场景

### 技术架构

列出：
- 编程语言
- 关键框架/库
- 依赖管理工具
- 其他基础设施

### 模型配置

必须包含：
- 推荐模型特性
- 完整的 YAML 配置示例
- 环境变量说明

### Python Client

必须包含：
- 导入语句
- 初始化方式
- 常用方法示例（chat、stream、list_models 等）

### REST API

必须包含：
- 基础调用示例（curl）
- 主要 API 端点说明
- 请求/响应格式

### Skill 编写指南

必须包含：
- 基本结构说明（frontmatter + 内容）
- 存放位置
- 示例代码

### 配置文件

必须包含：
- 完整配置示例
- 关键配置项说明
- 环境变量列表

### 项目文件结构

使用树形结构，包含：
- 主要目录
- 关键文件
- 简短注释说明

### 常用命令

使用代码块格式，按功能分组：
- 配置命令
- 安装命令
- 启动/停止命令
- Docker 命令（如适用）

### 故障排查

包含常见问题分类：
- 服务启动失败
- 连接失败
- 配置问题
- 性能问题

---

## 四、格式规范

### 表格

使用 Markdown 表格，标题行使用 `---` 分隔：

```markdown
| 列1 | 列2 |
|-----|-----|
| 内容1 | 内容2 |
```

### 代码块

- Bash 命令使用 `bash`
- Python 代码使用 `python`
- YAML 配置使用 `yaml`
- JSON 示例使用 `json`

### 链接

- 使用相对路径链接项目内文件
- 使用完整 URL 链接外部资源

### 强调

- 关键警告使用 ⚠️
- 重要提示使用加粗

---

## 五、检查清单

生成文档后，确保：

- [ ] 目录可点击跳转
- [ ] 代码块有正确的语言标识
- [ ] 链接路径正确
- [ ] 配置示例完整可运行
- [ ] 包含故障排查章节
- [ ] 包含安全使用建议
- [ ] 添加文档生成时间
