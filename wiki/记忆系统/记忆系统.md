# 记忆系统

> 记忆是 Agent 从"无状态问答"到"有经验的学习者"的关键基础设施。记忆不是简单的存储，而是能够自主组织、关联和进化的活知识。

## 为什么需要记忆

纯 RAG 每次查询从零开始——没有累积、没有综合、没有跨文档理解。Agent 需要记忆来：
- 保持长期交互的连贯性
- 跨 session 积累知识和经验
- 自主发现知识间的关联
- 持续精炼理解（而非每次重新推导）

## 记忆分类框架

| 维度 | 描述 | 示例 |
|------|------|------|
| **1D (Flat)** | 简单线性记忆 | 对话历史、提示词 |
| **2D (Planar)** | 结构化平面记忆 | 键值对、表格 |
| **3D (Hierarchical)** | 多层级树状记忆 | 文件系统、知识图谱 |

→ 详见 [[Memory-Survey]]

## 两种实践方案

### 学术方案：A-Mem（自主进化记忆）

基于 Zettelkasten，三大机制：
1. **笔记构建**：每条记忆 = 内容 + 关键词 + 标签 + 上下文描述 + embedding
2. **链接生成**：embedding 检索 + LLM 判断，建立语义关联
3. **记忆进化**：新记忆触发已有记忆的更新——**最独特的创新**

关键区别：不仅是检索（RAG），更是**存储层面的自主性**。

→ 详见 [[A-Mem]]

### 工程方案：文件系统记忆

- 文件系统 = 大小不限、天然持久化、Agent 可直接操作
- 压缩策略设计为可恢复（保留 URL/路径 → 内容可移除）
- 复述操控注意力（todo.md 强制 Agent 聚焦）
- 更侧重工程实用性而非语义丰富度

→ 详见 [[Manus-经验]]

## 记忆 vs 上下文

| | 上下文 | 记忆 |
|--|--------|------|
| 生命周期 | 单次对话 | 跨 session 持久 |
| 容量 | 有限（context window） | 理论无限 |
| 管理者 | LLM + 工程师 | LLM 自主 |
| 核心挑战 | 腐烂、焦虑、成本 | 组织、进化、检索 |
