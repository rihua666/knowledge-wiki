# 动态上下文发现 (Dynamic Context Discovery)

> Cursor 提出的模式：预先提供更少的上下文，让 Agent 按需自主提取，比静态预加载更高效。

## 核心内容

### 核心思想

随着模型能力提升，**预先提供更少的上下文**反而让 Agent 更容易自主按需提取相关上下文。

- **Static Context（静态上下文）**：始终被包含在上下文窗口中的信息
- **Dynamic Context Discovery（动态上下文发现）**：仅在需要时才加载的信息

### 五个具体实践

1. **长工具响应转文件** — 将大型 JSON 响应写入文件，Agent 用 `tail` 按需读取，避免上下文膨胀
2. **摘要时引用对话历史** — 用文件引用代替内联历史，减少 token 消耗
3. **支持 Agent Skills 开放标准** — 标准化的 Agent 技能定义
4. **仅加载所需 MCP 工具** — 不预加载所有 MCP 工具定义，按需发现和加载
5. **集成终端会话视为文件** — 终端输出持久化为文件，Agent 可以随时回溯

### 优势

- **Token 效率更高**：只将必要数据引入上下文
- **回复质量更好**：减少混淆和矛盾信息的干扰

## 关键要点

- 动态上下文是对静态上下文的补充，不是替代
- Cursor 为每个前沿模型单独优化 agent harness，但动态上下文是通用的
- 工具响应截断会导致数据丢失，写文件 + 按需读取是更好的方案
- 与 Anthropic 的 "just-in-time" 上下文策略理念一致
