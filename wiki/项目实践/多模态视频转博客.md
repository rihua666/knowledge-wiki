# 多模态视频转博客：Agent + 全模态理解工作流

> 用全模态模型（视频+图像+语音+文本联合推理）替代传统 ASR+LLM 流水线，实现技术视频到图文博客的自动化转换。
> 来源: 宝玉AI | 2026-05-06

## 核心要点

- **问题起源**：Karpathy 两年前想把 2h13m 的 tokenizer 教学视频自动转成博客，传统 ASR→LLM 流水线每步都丢信息（语气/画面/代码/架构图），效果不稳定 [原文]
- **全模态理解的突破**：Doubao-Seed-2.0-lite 可同时输入视频、图片、语音和文本做联合推理，不再需要把视频"拍扁"成纯文本 [原文]
- **四步工作流**（与 Agent 反复实践后的最佳实践）：
  1. **切片而非转写**：ffmpeg 自动切片长视频（>20min/50MB），下采样至 720p，每片保留完整多模态信息，并发处理后按时间线合并 [原文]
  2. **先生成素材，不憋终稿**：让模型先输出结构化写作素材（主题、章节、画面证据、术语、不确定点），再改写成博客 [原文]
  3. **反查视频挑关键帧**：模型基于文章内容反查视频，输出带 `reason` 字段的结构化 JSON——不只描述画面，还要解释"为什么这帧该放进文章" [原文]
  4. **确定性工具执行**：ffmpeg 截图、命名、保存、路径插入交给脚本，不交给模型 [原文]

## 关键设计：Skill 化原子能力

作者开发了 `doubao-multimodal` Skill，将多模态能力拆成 8 个原子化 task：

| task | 用途 | 保留画面 |
|------|------|---------|
| asr / asr-timestamp | 纯语音转写 | 否 |
| multispeaker-asr | 多说话人转写 | 否 |
| diarize | 说话人+时间段 | 否 |
| caption | 音视频整体描述 | 是 |
| video-timeline | 视频事件时间轴 | 是 |
| keyframe-extract | 技术博客配图 | 是 |
| understand | 自定义 prompt 理解 | 是 |

好处：原子化 task 可自由组合，换 prompt 和输出格式即可用于转写报告、竞品分析、课堂记录、游戏复盘等场景 [推断]

## 局限性

- 输入长度和大小仍有上限，长视频需切片降分辨率 [原文]
- 长输出稳定性需拆阶段设计，不能一步到位 [原文]
- 时间戳非帧级精确，高精度场景需取候选帧 [原文]
- 技术文章仍需人工审稿（API 版本、命令、事实判断）[原文]

## 方法论洞见

> "一个好用的多模态 Agent 工作流，不应该把所有事情都塞给模型。模型负责理解和决策，脚本负责稳定执行。" [原文]
