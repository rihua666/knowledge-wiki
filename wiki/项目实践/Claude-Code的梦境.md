# Claude Code 的梦境（Auto Dream）

> Anthropic 灰度上线 Auto Dream 功能——Agent 在后台自动整理记忆，类比人类睡眠中的记忆巩固。
> 来源: 公众号 | 2026-04-03

## 触发机制

自动触发需同时满足两个条件：
1. 距离上次整理已过 **24 小时**
2. 中间至少积累了 **5 个对话记录**

触发时在后台启动子代理，不影响当前对话。也可手动触发（`/dream`，但灰度阶段常报错）。

## 做梦四步骤

| 步骤 | 动作 |
|------|------|
| **Orient（定向）** | 读取整个记忆目录，了解当前状态 |
| **搜集信号** | 翻对话记录，找纠正、要求记住的、反复出现的、重要决策 |
| **巩固** | 信号与记忆逐个比对：合并重复、解决矛盾、修正相对日期、清理过时信息 |
| **修剪与索引** | 删除冗余和无用信息 |

社区案例：913 个对话 → 做梦 8-9 分钟 → MEMORY.md 从 280 行瘦身到 142 行，消除 3 条矛盾记录。

## Claude Code 四层记忆架构

| 层级 | 机制 | 类比 |
|------|------|------|
| **CLAUDE.md** | 手写指令文件，最高权限 | 显意识意图 |
| **Auto Memory** | 工作中自动记录 | 海马体（临时缓存） |
| **Session Memory** | 单次对话上下文 | 短期记忆 |
| **Auto Dream** | 后台记忆巩固 | 深度睡眠记忆重放 |

## 与人类记忆的同构

- 海马体 = Auto Memory（临时缓存）
- 大脑皮层 = 长期存储
- 深度睡眠回放 = Auto Dream 的记忆巩固
- AI 优势：原始对话记录（JSONL）永久保留在本地，享受遗忘好处但不丢失原始数据

## 安全约束

- 项目代码只读：只能写记忆文件，碰不了源码
- 同一项目同时只允许一个 Auto Dream 进程
